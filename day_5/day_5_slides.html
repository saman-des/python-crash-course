What is a Function?
The building blocks of code

Defining and Calling Functions

A function is a named block of code you can run whenever you want. Define it once with def, call it as many times as you need.

def say_hello():
#  ↑ keyword  ↑ name  ↑ () required
    print("Hello, world!")
#   ↑ indented body — runs when called

say_hello()  # ← call to run the function
⚠️ def creates the function. The function doesn't run until you call it with parentheses ().

Function Anatomy — Every Part Explained 🔍

def greet_user(name):
↑ keyword    ↑ function name    ↑ parameter (input)    ↑ colon required
    """Greet someone by name."""  ← docstring (optional but recommended)
    print(f"Hello, {name}!")  ← function body (indented 4 spaces)
greet_user("Alice")  ← calling the function with an argument
↑ function name    ↑ argument (value passed)
📋 Definition
  • •Starts with def
  • •Followed by function name
  • •Parentheses () — empty or with parameters
  • •Colon : at the end
  • •Body indented 4 spaces
📞 Calling
  • •Use the function name
  • •Add parentheses ()
  • •Pass arguments inside if needed
  • •Python runs the body

Parameters

Parameters — Functions That Accept Input

Parameters let you pass data into a function. The function can then use that data to do its job differently each time.

def greet_person(name):      # 'name' is a PARAMETER
    print(f"Hello, {name}!")

greet_person("Alice")        # "Alice" is an ARGUMENT
greet_person("Bob")          # "Bob" is an ARGUMENT
Parameter = variable in def
Argument = value in call

Multiple Parameters 🔢

SEPARATE WITH COMMAS
def describe_pet(pet_name, animal_type):
    print(f"I have a {animal_type} named {pet_name}.")

describe_pet("Fluffy", "cat")
describe_pet("Rex", "dog")
describe_pet("Nemo", "fish")
Arguments are matched to parameters left to right!
ORDER MATTERS
✅ Correct order:
describe_pet("Fluffy", "cat")
# → "I have a cat named Fluffy."
❌ Wrong order:
describe_pet("cat", "Fluffy")
# → "I have a Fluffy named cat." 😵

Return Values
Get data back from functions

Return — Sending Data Back

Functions that only print give you nothing to work with. Use return to send a value back to the caller!

def add_numbers(a, b):
    total = a + b
    return total       # ← sends 'total' back

answer = add_numbers(3, 5)    # ← store the returned value
print(answer)              # 8

# Can use directly too:
print(add_numbers(10, 20) + 5)  # 35
⚠️ After return, the function stops immediately. Any code after return is never reached!

Return Patterns 🔄

Early Return (Conditional)

def check_age(age):
    if age < 18:
        return "Minor"
    return "Adult"

print(check_age(15))   # Minor
print(check_age(25))   # Adult
No else needed — return exits the function!

Return Expression Directly

def square(n):
    return n * n    # No variable needed!

def c_to_f(c):
    return (c * 9/5) + 32

print(square(4))      # 16
print(c_to_f(100))   # 212.0
💡 A function without return automatically returns None.

Default Parameters

Default Values — Optional Arguments

Give parameters a default value in the definition. Callers can skip that argument — the default will be used instead!

def greet(name="friend"):
    print(f"Hello, {name}!")

greet("Alice")   # "Hello, Alice!"
greet()          # "Hello, friend!"
⚠️ Required parameters must come BEFORE optional ones!
10 / 18

Defaults Make Functions Flexible 🛠️

MIXING REQUIRED AND OPTIONAL
def make_smoothie(fruit, size="medium", ice=True):
    ice_txt = "with ice" if ice else "no ice"
    print(f"{size} {fruit} smoothie ({ice_txt})")

make_smoothie("mango")                    # medium, with ice
make_smoothie("banana", "large")          # large, with ice
make_smoothie("berry", "small", False)   # small, no ice
⚠️ GOTCHA: MUTABLE DEFAULTS

NEVER do this!

def bad(item, lst=[]):   # DANGER! Shared list!
    lst.append(item)
    return lst

Do this instead:

def good(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

Keyword Arguments
Name your values

Pass Arguments by Name

Use keyword arguments to specify which parameter gets which value. Order doesn't matter!

def describe_pet(name, animal, age):
    print(f"{name} is a {age}-year-old {animal}.")

# Keyword — order doesn't matter!
describe_pet(animal="dog", age=5, name="Rex")
💡 Keywords make code clearer, especially with many parameters or boolean flags!

Local vs Global Variables

Variables created inside a function are local. Variables created outside are global.

📍 Local
def demo():
    x = 5  # local
demo()
print(x)  # ERROR!
🌐 Global
y = 10  # global
def show():
    print(y)  # works!
