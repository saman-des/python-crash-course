Why Do We Need Loops? 🤔

Without a loop, repeating an action means copy-pasting the same code over and over. That's tedious, error-prone, and impossible to scale.

❌ Without a loop (5 friends)

print("Hello, Alice!")
print("Hello, Bob!")
print("Hello, Charlie!")
print("Hello, Diana!")
print("Hello, Eve!")
→ 100 friends = 100 lines? 😱
→

✅ With a loop (any number of friends)

friends = ["Alice","Bob","Charlie",
           "Diana","Eve", ...100 more]

for name in friends:
    print(f"Hello, {name}!")

→ Works for 5, 500, or 5 million friends!
🧠 A loop is Python's way of saying: "do this for every item" or "keep doing this until I tell you to stop".

The for Loop
Loop over items in a list

The for Loop — Building Blocks

A for loop takes each item from a list (or any sequence), puts it in a variable, and runs the indented block. Then it moves to the next item.

friends = ["Alice", "Bob", "Charlie"]
#          ↑ the list Python will loop over

for name in friends:
#    ↑         ↑
# loop var    the list
    print(f"Hello, {name}!")
# ↑ indented — runs every round
⚠️ The colon : after for ... in ...: is required. The body MUST be indented 4 spaces!

for Loop in Action 🔥

LOOP BODY CAN HAVE MANY LINES
fruits = ["apple", "banana", "mango"]

for fruit in fruits:
    upper  = fruit.upper()     # ← inside
    length = len(fruit)        # ← inside
    print(f"{upper} — {length} letters")  # ← inside

print("Done!")  # ← NOT indented — runs once
Everything at the same indent level is part of the loop — and all of it repeats every round!
LOOP + IF — THE POWER COMBO
scores = [85, 42, 91, 38, 77, 55]

for score in scores:
    if score >= 60:
        print(f"{score} → PASS ✅")
    else:
        print(f"{score} → FAIL ❌")

# Output:
# 85 → PASS ✅
# 42 → FAIL ❌
# 91 → PASS ✅  ... etc.
💡 Combining a loop with an if is the most common pattern in all of Python!

range()
Loop a fixed number of times

range() — Count for Me, Python!

When you don't have a list to loop over but you need to repeat something N times, use range() to generate the numbers automatically.

# range(5) generates: 0, 1, 2, 3, 4
for i in range(5):
    print(f"Round {i}")
# Prints: Round 0, Round 1, Round 2, Round 3, Round 4

# range(1, 6) generates: 1, 2, 3, 4, 5
for i in range(1, 6):
    print(f"5 × {i} = {5 * i}")
🔑 range(n) starts at 0 and stops before n. The stop number is never included!

range() in Action 🚀

MULTIPLICATION TABLE
for i in range(1, 11):
    print(f"7 × {i} = {7 * i}")
# 7×1=7, 7×2=14, ... 7×10=70
EVEN NUMBERS
for i in range(0, 11, 2):
    print(i, end="  ")
# 0  2  4  6  8  10
COUNTDOWN
for i in range(5, 0, -1):
    print(f"T minus {i}...")
print("Liftoff! 🚀")
RANGE + INDEX — enumerate() SHORTCUT
students = ["Alice","Bob","Charlie"]

# Manual way (range + index)
for i in range(len(students)):
    print(f"{i}: {students[i]}")

# Cleaner way — enumerate()
for i, name in enumerate(students, start=1):
    print(f"Seat {i}: {name}")
# Seat 1: Alice
# Seat 2: Bob
# Seat 3: Charlie
💡 enumerate(list, start=1) gives you the index AND the item at the same time — super handy!

The Accumulator Pattern

The most important loop pattern: create a bucket variable before the loop, fill it up a little each round, read the result after the loop ends.

🪣
bucket = 0
(before loop)
→
🔄
for each item:
bucket += item
→
📊
print(bucket)
(after loop)
prices = [120, 250, 75, 340]
total = 0                    # ← bucket starts empty
for price in prices:
    total += price           # ← fill it each round
print(f"Total: Rs. {total}")   # ← read it after

Accumulator — Four Flavours 🍦

Sum
total = 0          # bucket
for price in prices:
    total += price
Count matching items
passing = 0         # counter
for score in scores:
    if score >= 60:
        passing += 1
Build a new list
evens = []          # empty list
for n in numbers:
    if n % 2 == 0:
        evens.append(n)
Find the maximum
biggest = nums[0]   # start
for n in nums:
    if n > biggest:
        biggest = n
🧠 The three rules: (1) create the bucket before the loop   (2) update it inside the loop   (3) use it after the loop

while Loop
Loop until something changes

The while Loop

A while loop keeps repeating as long as a condition stays True. You use it when you don't know in advance how many rounds you'll need.

# Anatomy
while condition:
    # this block runs again and again
    # as long as condition is True

# Real example
countdown = 3

while countdown > 0:
    print(f"T minus {countdown}...")
    countdown -= 1   # ← MUST change the variable!

print("Liftoff! 🚀")
⚠️ You must change the variable inside the loop, otherwise the condition stays True forever — an infinite loop!

answer = input("Password: ")

while answer != "python":

💡 A for loop would not work here — you don't know how many wrong guesses the user will make. This is why while exists!

⚠️ The Infinite Loop — Most Common Mistake

DANGER — LOOPS FOREVER 💀
countdown = 5

while countdown > 0:
    print(countdown)
    # ← forgot: countdown -= 1 !!!

# prints 5, 5, 5, 5, 5 ... forever
# Press Ctrl+C to stop it!
❌ countdown never changes → condition never becomes False → loops forever!
SAFE — ALWAYS MAKE PROGRESS ✅
countdown = 5

while countdown > 0:
    print(countdown)
    countdown -= 1  # ← this line is essential!

# prints 5, 4, 3, 2, 1 → done
✅ Every round, countdown gets smaller → eventually hits 0 → condition becomes False → loop stops safely.
🆘 Stuck in an infinite loop? Press Ctrl + C in the terminal to force-stop it!

break & continue
Control inside a loop

break & continue — Remote Controls for Loops

Two keywords let you take manual control of what happens during a loop:

break
EXIT the entire loop immediately.
No more rounds. Done.
continue
SKIP the rest of this round.
Jump to the next item.
# break: stop when you find what you're looking for
for fruit in ["apple", "banana", "mango"]:
    if fruit == "banana": break
    print(fruit)    # only prints: apple
15 / 21

break & continue — Side by Side 🔬

break — EXITS the whole loop
data = [1, 2, -99, 4, 5]

for x in data:
    if x == -99:
        break        # stop everything
    print(x)

# Prints: 1  2
# (stops at -99, never reaches 4 or 5)
Use break to stop searching once you've found what you need.
continue — SKIPS this round, loop keeps going
data = [1, 2, -99, 4, 5]

for x in data:
    if x == -99:
        continue     # skip -99, go next
    print(x)

# Prints: 1  2  4  5
# (-99 is skipped, 4 and 5 still run)
Use continue to skip unwanted items while still processing the rest.
🧠 Memory trick: break = "I'm done, go home"  |  continue = "skip this one, keep going"

Nested Loops
A loop inside a loop

Nested Loops — Outer × Inner

A nested loop is a loop written inside another loop. For every single round of the outer loop, the inner loop runs completely from start to finish.

for i in range(1, 4):         # outer: 1, 2, 3
    for j in range(1, 4):     # inner: 1, 2, 3 (runs fully each outer round)
        print(f"({i},{j})", end=" ")
    print()            # newline after inner completes
# (1,1)(1,2)(1,3)
# (2,1)(2,2)(2,3)
# (3,1)(3,2)(3,3)   → 3×3 = 9 pairs total
🕐 Think of a clock: the minute hand (inner) completes a full circle every time the hour hand (outer) moves one step.

Nested Loops in Action — Patterns & Tables 🔢

MULTIPLICATION TABLE (3×3)
for row in range(1, 4):     # 1, 2, 3
    for col in range(1, 4): # 1, 2, 3
        print(f"{row * col}", end="\t")
    print()

# 1  2  3
# 2  4  6
# 3  6  9
STAR TRIANGLE
rows = 5

for i in range(1, rows + 1):
    for j in range(i):    # grows each row
        print("★", end=" ")
    print()

# ★
# ★ ★
# ★ ★ ★
# ★ ★ ★ ★
# ★ ★ ★ ★ ★
🪆 The inner loop runs i times each row — so row 1 has 1 star, row 2 has 2, etc. The outer loop controls which row we're on.
