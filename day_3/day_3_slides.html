What are Conditions? 🔀

Every day you make decisions based on conditions:

  • 🌧️If it's raining, take an umbrella
  • 🚦If the light is red, stop — else go
  • 🎓If score ≥ 60, you pass — else you fail
  • 🔑If password is correct, log in

Python uses conditions the same way — a question with a True or False answer determines what code runs.

🧠 Conditions turn data into decisions. This is where your programs start to feel intelligent!

if / else
The foundation

The if Statement — Two Paths

The if block runs only when the condition is True. The else block runs when it's False.

# Anatomy of an if/else
if condition:
    # runs when condition is TRUE
    print("Yes!")
else:
    # runs when condition is FALSE
    print("No!")

# Real example
age = 16

if age >= 18:
    print("You can vote! 🗳️")
else:
    print("Too young to vote.")
⚠️ The colon : at the end of if and else is required. The body must be indented with 4 spaces!

if / else in Action 💡

EXAMPLE 1 — PASSWORD CHECKER
password = input("Enter password: ")

if password == "python123":
    print("✅ Access granted!")
else:
    print("❌ Wrong password!")
💡 == compares (asks "are these equal?"). = assigns. Don't mix them up!
EXAMPLE 2 — INDENTATION RULES
score = 75

if score >= 60:
    print("You passed! 🎉")   # ← inside if
    print("Well done!")        # ← still inside if
else:
    print("You failed. 😔")    # ← inside else

print("Exam is over.")       # ← NOT indented, always runs
⚠️ Python uses indentation (4 spaces) to know what is "inside" a block. Wrong indentation = error!

elif
Multiple choices

elif — More Than Two Paths

elif stands for "else if". It adds more branches — Python checks them one at a time from top to bottom, and runs the FIRST one that is True.

score = 82

if   score >= 90:  print("A")   # 90–100
elif score >= 80:  print("B")   # 80–89  ← runs for 82
elif score >= 70:  print("C")   # 70–79
elif score >= 60:  print("D")   # 60–69
else:              print("F")   # below 60
🔑 You can have as many elif branches as you need. The else at the end is optional — it catches everything that didn't match.

elif in Action 🚦

Traffic Light

light = input("Colour? ").lower()

if light == "red":
    print("STOP! 🔴")
elif light == "yellow":
    print("Get ready... 🟡")
elif light == "green":
    print("GO! 🟢")
else:
    print("Unknown colour!")
💡 .lower() converts input to lowercase so "Red", "RED", "red" all match!

Shortcut: in with a list

month = int(input("Month (1–12): "))

if month in [12, 1, 2]:
    print("Winter ❄️")
elif month in [3, 4, 5]:
    print("Spring 🌸")
elif month in [6, 7, 8]:
    print("Summer ☀️")
else:
    print("Autumn 🍂")
✨ value in [a, b, c] is a clean way to check multiple values in one line!

and · or · not — In Depth 🔬

# AND — both must be True
is_member   = True
has_points  = False

if is_member and has_points:
    print("Free gift! 🎁")   # won't run — has_points is False

# OR — at least one True
if is_member or has_points:
    print("You get a discount!") # runs — is_member is True

# NOT — flips the boolean
is_raining = False
if not is_raining:
    print("Great day for a walk! ☀️")

🧠 and is strict — both must pass. or is generous — one is enough. not reverses the answer!

Truthy & Falsy Values 💡

Python doesn't need an actual True or False in a condition. Every value has a "truthiness"!

😴 FALSY — acts like False
bool(0)        # False — zero
bool(0.0)      # False — zero float
bool("")        # False — empty string
bool([])        # False — empty list
bool({})        # False — empty dict
bool(None)      # False — no value at all
✅ TRUTHY — acts like True
bool(1)        # True — any non-zero number
bool(-5)       # True — negative too!
bool("hello")  # True — any non-empty string
bool([1,2])    # True — non-empty list
bool(True)     # True — obviously!
name = input("Your name: ")
if name:    # True if user typed something, False if they just pressed Enter
    print(f"Hello, {name}!")

Nested if

Nested Conditions — Boxes Within Boxes

Sometimes a second question only makes sense after the first one is already True. Put an if inside an if!

age = int(input("Age: "))
has_id = input("Have ID? (yes/no): ")

if age >= 18:
    # Old enough — now check if they have ID
    if has_id == "yes":
        print("Welcome! ✅")
    else:
        print("Age OK, but no ID. ❌")
else:
    print("Too young. ❌")
💡 Each level of nesting adds 4 more spaces. If you find yourself 3+ levels deep, consider using and / or to flatten the logic instead!

Nested Conditions — ATM Simulation 🏧

balance = 5000
pin     = "4321"

entered = input("Enter PIN: ")

if entered == pin:
    print("PIN accepted ✅")
    amount = int(input("Withdraw Rs. "))

    if amount <= 0:
        print("Invalid amount.")
    elif amount > balance:
        print("Insufficient funds.")
    else:
        balance -= amount
        print(f"Dispensed Rs. {amount}")
        print(f"Balance: Rs. {balance}")
else:
    print("Wrong PIN. Card blocked! 🔒")
NEW SHORTHAND OPERATOR
# -= means subtract and update
balance -= amount
# same as:
balance = balance - amount

# Other shorthands:
x += 5   # x = x + 5
x *= 2   # x = x * 2
x //= 3  # x = x // 3
🪆 The inner if/elif/else only runs when the outer if is True. This is the power of nesting!

for & while — brief intro

Loops — Don't Repeat Yourself!

Without loops you'd have to copy-paste the same code hundreds of times. A loop tells Python "repeat this block of code".

❌ Without a loop

print("Hello 1")
print("Hello 2")
print("Hello 3")
... 97 more lines!

✅ With a loop

for i in range(100):
    print(f"Hello {i+1}")
Python has two types of loops: for (loop a known number of times or over a list) and while (loop until a condition changes).

The for Loop 🔄

Loop over a list

fruits = ["apple", "banana", "mango"]

for fruit in fruits:
    print(f"I love {fruit}!")

# Output:
# I love apple!
# I love banana!
# I love mango!
📌 fruit is a temporary variable — it changes to the next item every loop round.

range() — loop a fixed number of times

for i in range(5):          # 0,1,2,3,4
    print(i)

for i in range(1, 6):        # 1,2,3,4,5
    print(i)

for i in range(0, 11, 2):   # 0,2,4,6,8,10
    print(i, end=" ")

0 to n-1

The while Loop ♾️

COUNTDOWN EXAMPLE
countdown = 5

while countdown > 0:
    print(f"T minus {countdown}...")
    countdown -= 1   # ← MUST change the variable!

print("Liftoff! 🚀")
⚠️ Always make sure the condition eventually becomes False. If the variable never changes, you get an infinite loop that runs forever!
BREAK — EXIT EARLY
for num in range(1, 11):
    if num == 5:
        print("Found 5! Stopping.")
        break        # jump OUT of the loop
    print(num)
# prints: 1 2 3 4  Found 5! Stopping.
💡 break works in both for and while loops — it immediately exits the nearest loop.

for vs while — When to Use Each ⚡

🔄 for loop
for item in collection:
  • ✅You know HOW MANY times to repeat
  • ✅Looping over a list, string, range
  • 🎯Print multiplication table (10 times)
  • 🎯Sum every item in a list
♾️ while loop
while condition is True:
  • ✅You DON'T know how many times yet
  • ✅Keep going until something changes
  • 🎯Keep asking until correct password
  • 🎯Game loop — play until player quits
💡 Rule of thumb: for = "do this N times / for each item".   while = "keep doing this until something changes". When in doubt, for is usually the safer choice!
